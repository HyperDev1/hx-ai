## Session commands

| Command | Description |
|---------|-------------|
| `/hx` | Step mode — execute one unit at a time, pause between each |
| `/hx next` | Explicit step mode (same as `/hx`) |
| `/hx auto` | Autonomous mode — research, plan, execute, commit, repeat |
| `/hx quick` | Execute a quick task with HX guarantees without full planning overhead |
| `/hx stop` | Stop auto mode gracefully |
| `/hx pause` | Pause auto mode (preserves state, `/hx auto` to resume) |
| `/hx steer` | Hard-steer plan documents during execution |
| `/hx discuss` | Discuss architecture and decisions (works alongside auto mode) |
| `/hx rethink` | Conversational project reorganization |
| `/hx mcp` | MCP server status and connectivity |
| `/hx status` | Progress dashboard |
| `/hx widget` | Cycle dashboard widget: full / small / min / off |
| `/hx queue` | Queue and reorder future milestones (safe during auto mode) |
| `/hx capture` | Fire-and-forget thought capture (works during auto mode) |
| `/hx triage` | Manually trigger triage of pending captures |
| `/hx dispatch` | Dispatch a specific phase directly |
| `/hx history` | View execution history (supports `--cost`, `--phase`, `--model` filters) |
| `/hx forensics` | Full-access debugger for auto-mode failures |
| `/hx cleanup` | Clean up HX state files and stale worktrees |
| `/hx visualize` | Open workflow visualizer |
| `/hx export --html` | Generate self-contained HTML report |
| `/hx export --html --all` | Generate reports for all milestones |
| `/hx update` | Update HX to the latest version in-session |
| `/hx knowledge` | Add persistent project knowledge |
| `/hx fast` | Toggle service tier for supported models |
| `/hx rate` | Rate last unit's model tier (over/ok/under) |
| `/hx changelog` | Show categorized release notes |
| `/hx logs` | Browse activity logs, debug logs, and metrics |
| `/hx remote` | Control remote auto-mode |
| `/hx help` | Categorized command reference |

## CLI flags

| Flag | Description |
|------|-------------|
| `hx` | Start a new interactive session |
| `hx --continue` (`-c`) | Resume the most recent session |
| `hx --model <id>` | Override the default model |
| `hx --print "msg"` (`-p`) | Single-shot prompt mode (no TUI) |
| `hx --mode <text\|json\|rpc\|mcp>` | Output mode for non-interactive use |
| `hx --list-models [search]` | List available models and exit |
| `hx --web [path]` | Start browser-based web interface |
| `hx --worktree` (`-w`) `[name]` | Start session in a git worktree |
| `hx --no-session` | Disable session persistence |
| `hx --extension <path>` | Load an additional extension |
| `hx --version` (`-v`) | Print version and exit |
| `hx sessions` | Interactive session picker |
| `hx config` | Set up global API keys |
| `hx update` | Update HX to the latest version |

## Headless mode

`hx headless` runs commands without a TUI — designed for CI, cron jobs, and scripted automation.

```bash
hx headless              # run auto mode
hx headless next         # run a single unit
hx headless query        # instant JSON snapshot (~50ms, no LLM)
hx headless --timeout 600000 auto   # with timeout
hx headless new-milestone --context brief.md --auto
```

| Flag | Description |
|------|-------------|
| `--timeout N` | Overall timeout in milliseconds (default: 300000) |
| `--max-restarts N` | Auto-restart on crash (default: 3, set 0 to disable) |
| `--json` | Stream events as JSONL to stdout |
| `--model ID` | Override the model |
| `--context <file>` | Context file for `new-milestone` (use `-` for stdin) |
| `--auto` | Chain into auto-mode after milestone creation |

**Exit codes:** `0` = complete, `1` = error/timeout, `2` = blocked.

### `hx headless query`

Returns a JSON snapshot of the project state — no LLM session, instant response.

```bash
hx headless query | jq '.state.phase'      # "executing"
hx headless query | jq '.next'              # next dispatch action
hx headless query | jq '.cost.total'        # total spend
```

## MCP server mode

```bash
hx --mode mcp
```

Runs HX as a Model Context Protocol server over stdin/stdout, exposing all tools to external AI clients (Claude Desktop, VS Code Copilot, etc.).
